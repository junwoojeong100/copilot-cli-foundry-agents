# 모듈 4: Microsoft Agent Framework 워크플로우

## 학습 목표

이 모듈을 완료하면 다음을 할 수 있습니다:

- **Microsoft Agent Framework(MAF)**의 핵심 개념과 아키텍처를 이해한다
- **단일 에이전트**를 생성하고 Azure OpenAI와 연동한다
- **Handoff 패턴**으로 에이전트 간 작업 위임을 구현한다
- **GroupChat 패턴**으로 다중 에이전트 협업을 구현한다
- **커스텀 그래프 워크플로우**로 조건부 라우팅 파이프라인을 구축한다
- 비즈니스 요구사항에 맞는 **워크플로우 패턴을 선택**할 수 있다

---

## Microsoft Agent Framework 핵심 개념

### Semantic Kernel + AutoGen 통합

Microsoft Agent Framework(MAF)는 2025년에 발표된 차세대 에이전트 프레임워크로, 기존 **Semantic Kernel**과 **AutoGen**을 하나로 통합한 프레임워크입니다.

| 기존 프레임워크 | 강점 | MAF에서의 역할 |
|---|---|---|
| **Semantic Kernel** | 플러그인 시스템, 엔터프라이즈 통합 | 에이전트 런타임, 도구 연동 |
| **AutoGen** | 멀티 에이전트 대화, 자동화 패턴 | 오케스트레이션, 그룹챗 |

```
┌─────────────────────────────────────────────┐
│       Microsoft Agent Framework (MAF)       │
├─────────────────────────────────────────────┤
│  ┌───────────────┐   ┌───────────────────┐  │
│  │ Agent Runtime  │   │  Orchestrations   │  │
│  │ (SK 기반)      │   │  (AutoGen 기반)    │  │
│  └───────────────┘   └───────────────────┘  │
├─────────────────────────────────────────────┤
│  Azure OpenAI  │  Tools/Plugins  │  Memory  │
└─────────────────────────────────────────────┘
```

### 순차 파이프라인 워크플로우

MAF는 에이전트를 순차적으로 연결하여 파이프라인을 구성할 수 있습니다. 한 에이전트의 출력을 다음 에이전트의 입력으로 전달하고, 조건부 라우팅을 직접 코드로 구현하여 복잡한 멀티 에이전트 시스템을 직관적으로 설계할 수 있습니다.

```
[에이전트 A] ──→ [에이전트 B] ──→ [에이전트 C]
                    │
                    └──→ [에이전트 D]  (조건부 분기)
```

## 워크플로우 패턴 비교

| 패턴 | 설명 | 에이전트 수 | 통신 방식 | 적합한 사용 사례 |
|---|---|---|---|---|
| **단일 에이전트** | 하나의 에이전트가 모든 작업 처리 | 1 | 없음 | 간단한 Q&A, 단일 도메인 작업 |
| **Handoff** | 에이전트가 전문가에게 작업 위임 | 2~5 | 순차적 위임 | 고객 지원, 단계별 프로세스 |
| **GroupChat** | 여러 에이전트가 협업 토론 | 3~8 | 라운드로빈/토론 | 브레인스토밍, 다관점 분석 |
| **커스텀 순차** | 순차적으로 정의한 자유 워크플로우 | 제한 없음 | 조건부 라우팅 | 복잡한 파이프라인, ETL |

## 실습 1: 단일 에이전트 기본

> 📄 파일: `01_single_agent.py`

가장 기본적인 에이전트를 생성하고 실행합니다. Azure AI Foundry와 연동하여 단일 에이전트가 사용자 질문에 응답하는 패턴입니다.

### 핵심 구성요소

- **FoundryChatClient**: Azure AI Foundry 프로젝트와의 연결을 담당
- **Agent**: 지시사항(instructions)에 따라 동작하는 AI 에이전트
- **agent.run()**: 에이전트를 실행하고 결과를 반환

### 실행

```bash
cd module4-agent-framework
python 01_single_agent.py
```

### 예상 출력

```
=== 단일 에이전트 실행 ===
질문: Microsoft Agent Framework가 무엇인가요?

에이전트 응답:
Microsoft Agent Framework(MAF)는 Semantic Kernel과 AutoGen을 통합한
차세대 AI 에이전트 프레임워크입니다...
```

## 실습 2: Handoff 워크플로우

> 📄 파일: `02_handoff_workflow.py`

Handoff 패턴은 **접수 에이전트가 사용자 요청을 분석**한 뒤, 적절한 **전문가 에이전트에게 작업을 위임**하는 방식입니다. 고객 지원 시나리오에서 자주 활용됩니다.

### 워크플로우 구조

```
[사용자 요청]
     │
     ▼
[접수 담당 에이전트] ──분석──→ 기술 문의? ──→ [기술 지원 에이전트]
                              │
                              └─ 결제 문의? ──→ [결제 지원 에이전트]
```

### 핵심 구성요소

- **HandoffBuilder**: 에이전트 간 Handoff 규칙을 정의하는 빌더
- **with_start_agent()**: 사용자 입력을 최초로 받는 접수 담당 에이전트 지정
- **with_autonomous_mode()**: 사용자 입력 없이 자동으로 진행
- **전문가 에이전트**: 특정 도메인(기술/결제)에 특화된 응답 제공

### 실행

```bash
python 02_handoff_workflow.py
```

### 예상 출력

실제 출력은 `workflow.run()`이 반환한 최종 응답을 그대로 표시합니다(단계별 로그는 `agent-framework`의 직렬화 결과를 따릅니다).

```
=== Handoff 워크플로우 실행 ===
사용자 요청: 결제 오류가 발생했습니다. 카드 결제가 계속 실패하고 있어요.

--------------------------------------------------

[최종 응답]
<workflow.run()이 반환한 최종 응답 — 접수 담당이 결제 지원으로 handoff한 뒤 작성된 안내>

=== 실행 완료 ===
```

## 실습 3: GroupChat 워크플로우

> 📄 파일: `03_group_chat.py`

GroupChat 패턴은 **여러 에이전트가 하나의 주제에 대해 협업 토론**하는 방식입니다. 각 에이전트가 자신의 전문 관점에서 의견을 제시하며, 다관점 분석이 필요한 경우에 효과적입니다.

### 워크플로우 구조

```
        ┌──────────────────────┐
        │   GroupChat 오케스트레이터  │
        └──────────────────────┘
         ╱          │          ╲
[기획자]      [개발자]      [디자이너]
 Planner     Developer    Designer
```

### 핵심 구성요소

- **GroupChatBuilder**: 그룹챗 워크플로우를 구성하는 빌더
- **selection_func**: 다음 발화자를 선택하는 함수 (GroupChatState → str)
- **max_rounds**: 최대 토론 라운드 수
- **역할별 에이전트**: 기획자, 개발자, 디자이너 등 전문 역할

### 실행

```bash
python 03_group_chat.py
```

### 예상 출력

실제 출력은 `workflow.run()`이 반환한 최종 객체를 그대로 표시합니다(턴별 분리 출력이 아니며, 형식은 `agent-framework`의 직렬화 결과를 따릅니다).

```
=== GroupChat 워크플로우 실행 ===
주제: 모바일 앱 신규 기능 기획: AI 기반 개인화 추천 시스템을 도입하려고 합니다.
==================================================

[GroupChat 토론 결과]
<workflow.run()이 반환한 최종 결과 — 참여자별 발화가 누적된 대화 이력>

=== 실행 완료 ===
```

## 실습 4: 커스텀 순차 워크플로우

> 📄 파일: `04_custom_workflow.py`

커스텀 순차 워크플로우는 **에이전트를 순차적으로 연결하고 조건부 라우팅을 직접 구현**하여 복잡한 파이프라인을 구축합니다. 에이전트의 출력을 다음 에이전트의 입력으로 전달하며, 분기 로직을 직접 코드로 작성합니다.

### 워크플로우 구조: 콘텐츠 제작 파이프라인

```
[주제 분석 에이전트]
       │
       ├─ 기술 주제 ──→ [기술 작가 에이전트]──┐
       │                                    │
       └─ 일반 주제 ──→ [일반 작가 에이전트]──┤
                                            │
                                     [편집자 에이전트]
                                            │
                                     [최종 출력]
```

### 핵심 구성요소

- **Agent**: 각 단계를 담당하는 에이전트
- **agent.run()**: 에이전트를 실행하고 결과를 다음 단계로 전달
- **조건부 라우팅**: 주제 분석 결과에 따라 다른 경로로 분기

### 실행

```bash
python 04_custom_workflow.py
```

### 예상 출력

```
=== 커스텀 순차 워크플로우 실행 ===
입력 주제: Kubernetes 클러스터 최적화 전략

[1단계] 주제 분석 중...
  → 분석 결과: 기술 주제

[2단계] 기술 작가가 초안 작성 중...
  → 초안 작성 완료

[3단계] 편집자가 검토 및 개선 중...
  → 최종 편집 완료

=== 최종 콘텐츠 ===
Kubernetes 클러스터 최적화 전략...
```

## 워크플로우 패턴 선택 가이드

프로젝트 요구사항에 따라 적절한 패턴을 선택하세요:

```
시작
 │
 ├─ 에이전트가 1개면 충분한가?
 │   └─ Yes → 단일 에이전트
 │
 ├─ 작업을 전문가에게 위임해야 하나?
 │   └─ Yes → Handoff 패턴
 │
 ├─ 여러 관점의 협업 토론이 필요한가?
 │   └─ Yes → GroupChat 패턴
 │
 └─ 복잡한 조건부 로직이 필요한가?
     └─ Yes → 커스텀 순차 워크플로우
```

| 고려사항 | 단일 에이전트 | Handoff | GroupChat | 커스텀 순차 |
|---|---|---|---|---|
| 구현 복잡도 | ⭐ | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 유연성 | ⭐ | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 비용 (API 호출) | 낮음 | 중간 | 높음 | 가변적 |
| 디버깅 용이성 | 쉬움 | 보통 | 어려움 | 보통 |

## 핵심 정리

1. **Microsoft Agent Framework**는 Semantic Kernel + AutoGen을 통합한 차세대 에이전트 프레임워크이다
2. **FoundryChatClient**로 Azure AI Foundry 프로젝트에 연결하고 **Agent**로 에이전트를 생성한다
3. **단일 에이전트**는 가장 간단한 패턴으로, 하나의 에이전트가 모든 작업을 처리한다
4. **Handoff 패턴**은 접수 에이전트가 전문가에게 작업을 위임하는 구조이다
5. **GroupChat 패턴**은 여러 에이전트가 협업하여 다관점 분석을 수행한다
6. **커스텀 그래프**는 조건부 라우팅 등 복잡한 파이프라인 구현에 적합하다
7. 비즈니스 요구사항(복잡도, 비용, 유연성)에 따라 적절한 패턴을 선택해야 한다
