# Repository Guidelines

These conventions apply to all contributors and AI coding agents working in this repository.

## Git Commit Messages

- **Always write commit messages in English.**
- Use clear, imperative mood (e.g., "Add module 2 MCP server example", "Fix typo in README").
- Keep the subject line concise (ideally ≤ 72 characters) and add a body when additional context is helpful.

## Pull Requests

- **All pull requests must target the `main` branch.**
- Use descriptive PR titles written in English.
- Summarize changes, motivation, and testing notes in the PR description.
